## Design Context

### Users
Canopy serves all audiences through progressive disclosure:
- **PL researchers** exploring CRDT + incremental parsing internals (need transparency)
- **Developer tool builders** extending language tooling (need precision and speed)
- **Showcase visitors** at conferences and demos (need to be impressed and intrigued)
The interface scales from minimal (newcomer) to full workbench (researcher) via configurable panels.

### Brand Personality
**Elegant, Thoughtful, Deep** — like a well-crafted academic tool where beauty emerges from structure. Not flashy, but reveals layers of sophistication on closer inspection. The kind of tool that makes you think "someone who deeply understands this domain built this."

### Aesthetic Direction
- **Visual tone**: Dark, focused, precise. Deep navy backgrounds with restrained purple accents. Typography-driven hierarchy.
- **References**: Zed Editor (clean dark theme, great typography), Dark/Luna (code meets visual representation, node-based), Strudel (live coding aesthetic, creative depth, beautiful dark mode with expressive color)
- **Anti-references**: Generic SaaS (no blue-purple gradients, no startup aesthetic), Toy/playground (no childish colors, no excessive animation — should feel serious and capable)
- **Theme**: Dark mode primary. The palette should feel like a night sky with constellations — deep, calm, with precise points of color.

### Design Principles
1. **Structure reveals meaning** — the visual design should make the AST structure legible at a glance. Color, spacing, and nesting communicate relationships before the user reads labels.
2. **Progressive disclosure** — start clean and focused, reveal depth on demand. Every panel toggle should feel like opening a drawer, not being overwhelmed.
3. **Typography carries weight** — Inter for UI (clean, professional), JetBrains Mono for code (precise, technical). The contrast between UI and code fonts should create clear zones.
4. **Color is semantic, not decorative** — every color means something. Purple = structure/accent, syntax colors follow consistent language semantics. No color without purpose.
5. **Calm confidence** — the interface should feel solid and trustworthy, never frantic or busy. Subtle transitions, restrained hover states, generous whitespace.
