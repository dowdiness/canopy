Editor Landscape vs. Your Priorities

Open-source editors evaluated against: Text UX → Structural Editing → Extensibility → Collaboration

Text / Code Editors

CM6

CodeMirror 6

Text UX: ★★★★★ — Built for code. Syntax highlighting, keybindings, multi-cursor, line numbers.

Structural: ★★☆☆☆ — Text-only model. No native block/node structure.

Extensibility: ★★★★★ — Modular architecture, Lezer parser system, rich plugin ecosystem.

Collab: ★★★★☆ — Built-in @codemirror/collab extension. Maps cleanly to OT/CRDT.

~150KB gzip. Used by: Replit, Observable, Chrome DevTools.

MO

Monaco Editor

Text UX: ★★★★★ — VS Code's editor. Full IDE experience out-of-box.

Structural: ★☆☆☆☆ — Purely text. No structural editing support.

Extensibility: ★★★☆☆ — VS Code extension API, but tightly coupled architecture.

Collab: ★★☆☆☆ — No built-in collab. Must build custom integration.

~2MB gzip. Heavy, hard to embed. Designed for full-page IDE.

Structured / Rich-Text Editors

PM

ProseMirror

Text UX: ★★★☆☆ — Rich text, not code. No syntax highlighting, no line numbers.

Structural: ★★★★★ — Schema-driven document model. Custom node types. Block manipulation.

Extensibility: ★★★★★ — Plugin system, custom schemas, decorations, node views.

Collab: ★★★★★ — prosemirror-collab + Step-based OT. Built for it.

~100KB gzip. Used by: Notion, Atlassian, NY Times.

LX

Lexical (Meta)

Text UX: ★★★☆☆ — Rich text focus. Code blocks via plugin, not native.

Structural: ★★★★☆ — Custom node classes. Tree-based model. React-native.

Extensibility: ★★★★☆ — Node/plugin system. Growing ecosystem, less mature than PM.

Collab: ★★★☆☆ — Yjs integration exists, but less battle-tested than PM.

~30KB gzip. Newer (2022). React-first but has vanilla JS mode.

TT

Tiptap

Text UX: ★★★☆☆ — Built on ProseMirror. Same rich-text strengths/weaknesses.

Structural: ★★★★☆ — Inherits PM's schema system with friendlier API.

Extensibility: ★★★★☆ — Extension kit wrapping PM plugins. Easier but less flexible.

Collab: ★★★★☆ — Yjs + Hocuspocus. Collab-as-a-service model (paid tier).

Built on ProseMirror. Higher-level API, React/Vue wrappers. Open core.

Emerging / Specialized

SL

Slate

Text UX: ★★☆☆☆ — Minimal defaults. You build everything.

Structural: ★★★★☆ — JSON document model. Flexible node/mark system.

Extensibility: ★★★☆☆ — Flexible but fragmented. Less plugin ecosystem.

Collab: ★★☆☆☆ — Possible but no built-in support. Slate-yjs exists but rough.

React-only. History of breaking changes. v0.x still.

LZ

Lezer (Parser System)

Not an editor — CodeMirror 6's incremental parser. Worth mentioning because it's architecturally similar to your MoonBit ImperativeParser. Could potentially share parsing infrastructure or inform your parser's design for new languages.

Incremental, error-recovering, grammar-driven. Same author as CM6 & PM (Marijn Haverbeke).
